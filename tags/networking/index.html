Another SSH trick

Ever have a machine you can only ssh into through another machine? It’s a very common situation in the Technion. Here’s one way to get around it: Assume you can directly ssh into alpha, and from alpha you can ssh into beta. Have the following code in your ~/.ssh/config: Host beta Hostname 1.2.3.4 # IP Address of beta ProxyCommand ssh alpha nc -w 1 %h %p This requires you to have nc (netcat) installed on alpha....

Using Jabber from within the Technion

A very neat find for those of you who want to use Jabber from within the Technion, but with your client of choice rather than a web-based one: Many Jabber servers, including Google Talk, support using Port 443 over SSL. Since the Technion does not block outbound SSL connections, this will work there as well.
